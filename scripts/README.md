# Скрипты для работы с Kwork и LLM

## Назначение скриптов

### `test_llm_integration.py`
Запускает тестирование интеграции LLM с тестовыми заказами.

### `wait_and_test.py`
Ожидает завершения загрузки модели и автоматически запускает тесты.

## Запуск тестов

1. Дождитесь завершения загрузки модели (проверьте статус в терминале).
2. Запустите тесты:
   ```bash
   python -m scripts.test_llm_integration
   ```
   Или дождитесь автоматического запуска через `wait_and_test.py`.

## Анализ результатов

1. Проверьте логи в `test_llm_integration.log`.
2. Убедитесь, что все тестовые заказы обработаны.
3. Проверьте корректность решений LLM в логах.
4. Убедитесь, что в уведомлениях (если настроены) присутствует анализ ИИ.

## Настройка параметров LLM

Параметры LLM можно изменить в `src/kwork/order_processor.py`:
- `temperature` (по умолчанию 0.2) - контролирует случайность ответов
- `max_tokens` (по умолчанию 300) - максимальное количество токенов в ответе
- `n_ctx` (по умолчанию 2048) - размер контекстного окна
- `n_threads` (по умолчанию 4) - количество потоков для обработки
