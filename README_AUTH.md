# Аутентификация и авторизация

## Обзор

Система аутентификации и авторизации использует JWT (JSON Web Tokens) для безопасной аутентификации пользователей и управления доступом. Аутентификация основана на OAuth2 с использованием паролей и токенов.

## Установка зависимостей

```bash
pip install -r requirements.txt
```

## Настройка окружения

Скопируйте файл `.env.example` в `.env` и настройте переменные окружения:

```bash
cp .env.example .env
```

### Основные настройки

- `SECRET_KEY` - Секретный ключ для подписи JWT токенов
- `JWT_ALGORITHM` - Алгоритм шифрования (по умолчанию: HS256)
- `ACCESS_TOKEN_EXPIRE_MINUTES` - Время жизни access токена (минуты)
- `REFRESH_TOKEN_EXPIRE_DAYS` - Время жизни refresh токена (дни)
- `ALLOWED_HOSTS` - Разрешенные хосты (через запятую)

## Запуск сервера

```bash
# Запуск в режиме разработки с автоматической перезагрузкой
python -m src.main --reload

# Запуск с пользовательскими настройками хоста и порта
python -m src.main --host 0.0.0.0 --port 8000
```

## API Endpoints

### Аутентификация

- `POST /api/v1/auth/token` - Получение access и refresh токенов
- `POST /api/v1/auth/refresh` - Обновление access токена
- `POST /api/v1/auth/register` - Регистрация нового пользователя
- `POST /api/v1/auth/verify-email` - Подтверждение email
- `POST /api/v1/auth/request-password-reset` - Запрос сброса пароля
- `POST /api/v1/auth/reset-password` - Сброс пароля
- `POST /api/v1/auth/logout` - Выход из системы
- `GET /api/v1/auth/me` - Получение информации о текущем пользователе

## Роли и разрешения

### Встроенные роли

- `user` - Обычный пользователь (базовые права)
- `admin` - Администратор (полный доступ)
- `review:read` - Просмотр отзывов
- `review:write` - Создание и редактирование отзывов

### Управление ролями

Роли и разрешения можно настраивать через админ-панель или напрямую в базе данных.

## Примеры запросов

### Получение токена

```bash
curl -X 'POST' \
  'http://localhost:8000/api/v1/auth/token' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'grant_type=password&username=user1&password=password123&scope=user'
```

### Обновление токена

```bash
curl -X 'POST' \
  'http://localhost:8000/api/v1/auth/refresh' \
  -H 'accept: application/json' \
  -H 'Authorization: Bearer your_refresh_token_here'
```

### Защищенные запросы

```bash
# Получение информации о текущем пользователе
curl -X 'GET' \
  'http://localhost:8000/api/v1/auth/me' \
  -H 'accept: application/json' \
  -H 'Authorization: Bearer your_access_token_here'
```

## Безопасность

- Все пароли хранятся в хешированном виде с использованием bcrypt
- Access токены имеют короткое время жизни (по умолчанию 15 минут)
- Refresh токены имеют более длительное время жизни (по умолчанию 7 дней)
- Токены подписываются с использованием секретного ключа
- Поддерживается HTTPS (настоятельно рекомендуется для продакшена)

## Развертывание

1. Убедитесь, что все зависимости установлены
2. Настройте переменные окружения в `.env`
3. Запустите миграции базы данных:
   ```bash
   alembic upgrade head
   ```
4. Запустите сервер:
   ```bash
   uvicorn src.main:app --host 0.0.0.0 --port 8000
   ```

## Устранение неполадок

### Ошибка аутентификации
- Убедитесь, что имя пользователя и пароль верны
- Проверьте, что пользователь активен и верифицирован
- Убедитесь, что токен не истек

### Ошибки доступа
- Проверьте, что у пользователя есть необходимые роли и разрешения
- Убедитесь, что токен содержит нужные scope
